Erros & Status HTTP

A API retorna erros em formato padronizado com timestamp, status, error, message e path.

2xx — Sucesso
201 Created Pagamento criado com sucesso.
4xx — Erro do cliente
400 Bad Request Campos inválidos ou ausentes. Leia message para detalhes.
401 Unauthorized Header x-api-key ausente ou chave inválida.
403 Forbidden Tenant inativo. Entre em contato com o suporte.
409 Conflict idempotencyKey já está em processamento ativo.
429 Too Many Requests Limite excedido. Ver Rate Limiting.
5xx — Erro do servidor
500 Server Error Erro inesperado. Abra uma issue com path e timestamp.

Exemplo de Resposta de Erro

{ "timestamp": 1740161400000, "status": 409, "error": "Conflito de Idempotencia", "message": "Pagamento já está em processamento ativo.", "path": "/v1/payments" }

Todos os erros incluem um timestamp Unix em milissegundos para correlação com logs internos. Inclua esse valor ao reportar problemas.
