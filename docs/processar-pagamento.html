Processar Pagamento Novo

Cria uma transação e roteia automaticamente para o gateway ideal com base na moeda e método de pagamento.

POST /v1/payments Requer x-api-key

Parâmetros

CampoReq.Descricao
idempotencyKey sim Identificador único por transação. Use UUID v4 vinculado ao ID do pedido no seu sistema.
amount sim Valor em ponto flutuante. Mínimo: 0.01.
currency sim Código ISO 4217: BRL, USD, EUR. Define o gateway que o SmartRouter selecionará.
paymentMethodRequest sim Objeto com type (pix, boleto, card_token) e token — obrigatório somente para cartão.
customer sim Objeto com id, email e document (CPF/CNPJ) do pagador.
metadata.gateway não Força um gateway específico: STRIPE, MERCADO_PAGO, PAYPAL, MOCK. Sobrescreve o SmartRouter.
